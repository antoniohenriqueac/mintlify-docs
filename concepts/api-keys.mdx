As **API Keys** são credenciais que autenticam e autorizam acesso à API ConnectVets Notes. Cada chave possui permissões e limites específicos baseados no seu plano.

## Como Funcionam

<Steps>
  <Step title="Solicitação">
    Contate nossa equipe via [WhatsApp (+55 31 8883-5141)](https://wa.me/5531888835141) para solicitar acesso
  </Step>
  <Step title="Configuração">
    Nossa equipe criará suas credenciais com limites adequados ao seu uso
  </Step>
  <Step title="Entrega">
    Você receberá sua API Key e instruções de uso por canal seguro
  </Step>
  <Step title="Integração">
    Use a chave no header `X-API-KEY` em todas as requisições
  </Step>
</Steps>

## Tipos de API Keys

### Produção
- **Ambiente**: `https://api.connectvets.com.br/v1`
- **Uso**: Aplicações em produção
- **Rate Limiting**: Conforme plano contratado
- **Suporte**: Prioritário

### Teste/Desenvolvimento  
- **Ambiente**: `https://api-dev.connectvets.com.br/v1`
- **Uso**: Desenvolvimento e testes
- **Rate Limiting**: Limitado para testes
- **Suporte**: Via Discord

## Configuração

### Headers Obrigatórios

```http
X-API-KEY: sua_api_key_aqui
Content-Type: application/json
```

### Exemplo de Uso

```javascript
const apiKey = process.env.CONNECTVETS_API_KEY;

const response = await fetch('https://api.connectvets.com.br/v1/notes', {
  method: 'GET',
  headers: {
    'X-API-KEY': apiKey,
    'Content-Type': 'application/json'
  }
});
```

## Troubleshooting

### Erro 401: Unauthorized

```json
{
  "error": "unauthorized",
  "message": "Invalid or missing API key"
}
```

**Soluções:**
- Verifique se o header `X-API-KEY` está presente
- Confirme que a chave está correta
- Verifique se a chave não expirou

### Erro 403: Forbidden

```json
{
  "error": "forbidden", 
  "message": "Insufficient permissions for this endpoint"
}
```

**Soluções:**
- Verifique se sua chave tem permissões para o endpoint
- Contate nossa equipe para upgrade de plano

### Erro 429: Rate Limit

```json
{
  "error": "too_many_requests",
  "message": "Rate limit exceeded",
  "retry_after": 60
}
```

**Soluções:**
- Implemente retry com backoff exponencial
- Considere upgrade para plano com limite maior
- Otimize frequência de requisições
